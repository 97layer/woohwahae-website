# Content Flywheel Roadmap

Historical archive only.
This note is kept for continuity and should not be treated as active runtime authority or default read-path material.

## Intent

Turn `97layer` into the raw intake and thinking layer, then reuse that source
across `woosunhokr`, `woohwahae`, blog, Shorts, and longer-form media without
collapsing everything into one tone.

This is not a “copy once, paste everywhere” model.
It is a **source -> analysis -> worldview -> adaptation -> publish -> learn**
loop.

See also:

- `docs/source_intake_operating_model.md`

## Core Structure

### Layer 1. Raw Intake — `97layer`

`97layer` is the primary source bucket.

What accumulates here:

- original thoughts
- work logs
- behind-the-scenes brand-building notes
- external sources worth analyzing
- reactions, questions, fragments, unfinished observations
- outside links and texts that should be interpreted before they become outbound content
- broad-domain intake across development, beauty, finance, humanities, systems, and daily life

Role:

- rawest seat
- first observation
- highest volume
- lowest polish

This is where the system learns what the founder keeps returning to.

## Layer 2. Worldview Distillation

Raw inputs should not jump straight into polished publishing.

The system should repeatedly extract:

- recurring themes
- repeated questions
- preferred metaphors
- tone patterns
- worldview anchors
- domain splits: life / system / beauty / brand / culture

This becomes the evolving worldview corpus.

Suggested passes:

1. first pass: simple tagging
2. second pass: cluster recurring ideas
3. third pass: promote stable worldview patterns

## Layer 3. Account Translation

The same source does not publish in the same form everywhere.

### `97layer`

- raw diary
- process
- observation
- maker note
- unfinished but alive

### `woosunhokr`

- the same thought translated into beauty/craft practice
- more refined
- closer to service, transformation, aesthetic judgment
- often the “미용사의 단상” version of a broader founder thought

### `woohwahae`

- the same base translated into public brand language
- most edited
- strongest signal-to-noise ratio
- may include “how we do it” or public-facing distilled philosophy
- should also be able to hold slower magazine-like pieces once enough source has been reduced

## Layer 4. Format Flywheel

One source can move across formats with different compression levels.

### Short -> Long

- Thread note
- short caption
- voice memo
- Telegram note

becomes:

- blog post
- essay
- YouTube long-form topic
- brand note

### Long -> Short

- blog or essay
- recorded reflection
- longer interview

becomes:

- Shorts
- Reels
- Threads fragments
- Telegram packets

Goal:

- one source, multiple surfaces
- but always rewritten by seat, not duplicated blindly

## External Source Use

External content should first land in the `97layer` intake side.

Why:

- outside material should influence the worldview, not overwrite it
- imported material needs interpretation before publication
- founder taste should stay the editor, not the feed

Recommended flow:

1. collect external source
2. summarize it
3. map it to existing worldview themes
4. decide whether it strengthens, contradicts, or expands the current spine
5. only then adapt it into outbound content

Later approval model:

1. source lands in `97layer`
2. the system proposes route candidates
3. founder approves or rejects in Telegram
4. only approved routes open the publish corridor

## Practical Metadata

Every usable content unit should eventually carry at least:

- `source_account`
- `theme`
- `tone_level`
- `format`
- `domain`
- `origin_type` (`founder`, `external`, `runtime`, `legacy`)
- `reuse_candidates`

This makes later adaptation and retrieval much easier.

## Roadmap Phases

### Phase 1. Source Capture

- keep feeding `97layer`
- store raw notes, reactions, and observations
- do not over-polish too early

### Phase 2. Theme Extraction

- detect recurring worldview themes
- build a stable tag vocabulary
- separate raw signal from promoted worldview

### Phase 3. Account Mapping

- define how a raw `97layer` note becomes:
  - `woosunhokr`
  - `woohwahae`
- keep the shared essence, change the finish level
- keep account destination explicit so posts do not drift between raw / refined / polished seats by accident

### Phase 4. Format Reuse

- connect Threads, blog, Shorts, long-form video, and Telegram
- allow short-to-long and long-to-short loops

### Phase 5. Closed Learning Loop

- observe what resonates
- update worldview clusters
- refine translation rules
- keep founder taste as final editor

## Immediate Decision

For now:

- keep using `97layer` as the raw intake hub
- let external sources land there first
- treat `woosunhokr` and `woohwahae` as translation layers, not separate disconnected systems
- design later for one-source-multi-use circulation
- do not treat one source artifact as auto-reusable everywhere; translate by account and approve before publish
