# Lineage

## Source Of Direction

Layer OS does not duplicate bootstrap-era constitutional source documents.

The new workspace inherits direction that was distilled during bootstrap.
This note exists only to explain lineage, not to create a live dependency.

Two legacy source texts matter most in that lineage:

- `the_origin`
  - subtraction over accumulation
  - rhythm over acceleration
  - the center is revealed by removing overlays
- `sage_architect`
  - calm and exact voice
  - lens before sermon
  - structure without theater

## Inheritance Rule

The new runtime inherits high-level direction only:

- essence over noise
- founder-led authority
- low-friction surface
- structural clarity

It does not inherit:

- old agent taxonomies
- Python runtime structure
- low-level visual token tables
- execution-era implementation mechanics

## Port Map

The legacy essence was intentionally split across the new constitution instead of being copied as a single authority block.

- `constitution/origin.md`
  - subtraction, zero-point thinking, rhythm, and clarity
- `constitution/brand.md`
  - one shell, quiet authority, and anti-theater posture
- `constitution/soul.md`
  - interpretive continuity without creating a second state authority
- `constitution/surface.md`
  - low-noise surface rules and anti-fragmentation
- `constitution/voice.md`
  - low-temperature language, sentence discipline, and anti-inflation rules
- `constitution/orchestra.md`
  - staff-aide operating posture and lane-first collaboration

If a future change needs the spirit of the old texts, port the invariant into one of the constitutional documents above.
Do not restore the legacy files as live runtime authority.

## Future Rule

As Layer OS matures, even this lineage note should become thinner.
Active operation should not depend on the previous system.
