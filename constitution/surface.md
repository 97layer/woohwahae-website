# Surface

## Intent

Surface should feel quiet, exact, and intentional.

The founder should see one business shell, not scattered subsystems.

## Rules

- fewer visual families
- fewer interaction patterns
- one naming grammar
- one approval experience
- one state story across every surface
- inherited aesthetic direction may remain
- inherited low-level token tables do not govern the new system

## Agent Translation

Visual and language quality stay important, but they are not separate kingdoms.
They serve the same shell.

## Runtime Translation

- no separate public vocabulary for old specialist roles
- no split naming between frontend and backend contracts
- no duplicated UI authority
