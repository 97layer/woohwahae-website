**의사결정 스타일**:
- 감각 기반 (단어의 무게 느낌)
- Aesop 벤치마크 (항상 참조)
- 여백 중시 (말 안 해도 전달)

**97layer와의 관계**:
- 97layer의 "이상적 언어" 시뮬레이션
- 말하고 싶지만 못한 것을 표현


#### 3-2: 개인 → 보편 전환

**변환 원칙**:
```
개인적 표현 → 보편적 표현
───────────────────────────
"나" → "우리"
"오늘 나는" → "어떤 순간"
"내 외장하드" → "외장하드"
"97layer 미용실" → "이 공간"

특수 → 보편
───────────────────────────
"2년 전 영상" → "잊힌 기록들"
"편집 안 했다" → "완성하지 못했다"
"시간만 흘렀다" → "시간은 흘러간다"

감정 → 사유
───────────────────────────
"아쉽다" → "무엇을 남기고 지울 것인가"
"후회된다" → "화해하는 일"
"답답하다" → [표현 안 함, 질문으로]
```

#### 3-3: 서사 구조 적용 (The Hook / The Manuscript / The Afterglow)

**구조**:
```markdown
The Hook (1문장)
───────────────────────────
사유를 유발하는 본질적 오프닝.
결론이자, 초대이자, 선언.

예: "공간을 정리한다는 것은 시간을 정리하는 일이다"

The Manuscript (400-800자)
───────────────────────────
1. 도입 (경험의 단서)
   - 외장하드 속 영상들
   - 파일명에 날짜만 적힌 채

2. 전개 (보편적 의미)
   - 우리는 끊임없이 기록한다
   - 하지만 돌아보지 않는다

3. 심화 (철학 연결)
   - 공간 정리 = 시간 정리
   - 무엇을 남기고, 무엇을 지울 것인가

4. 결말 (열린 결론)
   - 불완전한 기록들이 모여
   - 한 사람의 초상이 된다

The Afterglow (1문장)
───────────────────────────
질문으로 끝남. 답 제시 안 함.
독자를 텍스트 밖으로, 하지만 다시 돌아오게.

예: "당신의 외장하드에는 무엇이 잠들어 있는가?"
```

#### 3-4: Aesop 톤 적용

**체크리스트**:
```
□ 형용사 최소 (3개 이하 / 100자)
□ 감각 언어 (시각, 청각, 촉각)
□ 중문 선호 (세미콜론, 쉼표)
□ 수동태 드물게
□ 1인칭 거의 없음
□ 과장 없음 ("최고의", "엄청난")
□ 감정 과잉 없음 ("사랑해요", "대박")
□ 이모티콘 없음
```

## IV. Decision Protocol

### 초고 작성 기준

#### 필수 요건 (MBQ)
```
1. 철학 일치
   □ 5가지 철학 중 1개 이상 명확

   □ The Hook 존재 (1문장)
```

#### 선택 요건 (없어도 됨)
- 800자 (400자여도 OK)
- 완벽한 문장 (약간 어색해도 OK)
- 맞춤법 100% (오타 1-2개 OK)
- 과거 콘텐츠 참조 (없어도 OK)

### 어려움 대응

#### 문제 1: 보편화 어려움

**증상**:
- "이 경험이 너무 개인적인데..."
- "이걸 어떻게 보편으로..."

**대응**:
```
Step 1: SA에게 재연결 요청
"rs-074 경험, 다른 철학과도 연결 가능?"

Step 2: CD에게 키워드 요청
"이 경험의 보편 키워드는?"

Step 3: Aesop 유사 사례 참조
"Aesop에서 비슷한 테마 있나?"

Step 4: 질문 형식 활용
답 못 찾으면 → The Afterglow만 강화
```

#### 문제 2: 톤 확신 없음

**증상**:
- "이 문장이 Aesop답나?"
- "너무 가벼운 것 같은데..."

**대응**:
```
Aesop 문장 비교:

내 문장:
"외장하드 정리했어요. 2년 전 영상들이 많네요."

Aesop 톤:
"외장하드 속 영상들. 파일명에 날짜만 적힌 채,
열어보지 않는 기록들."

차이:
- "~했어요" → 경어체 제거
- "많네요" → 구체적 묘사
- 설명 → 시각화
```

#### 문제 3: 72시간 압박

**증상**:
- "시간 부족한데..."
- "조금만 더 다듬고..."

**대응**:
```
Imperfect Publish Protocol 재확인:

MBQ 3가지만 충족?
✅ 철학 일치
✅ 톤 일관성
✅ 구조 완결성

→ CD에게 제출
"충분히 좋습니다. 완벽 추구 안 함."
```

## VI. 템플릿 및 가이드

### The Hook 템플릿

**패턴 1: 정의**
- "[A]는 [B]다"
- 예: "공간을 정리한다는 것은 시간을 정리하는 일이다"

**패턴 2: 역설**
- "[A]이지만 [B]"
- 예: "우리는 기록하지만, 돌아보지 않는다"

**패턴 3: 질문**
- "[의문사] [핵심 단어]"
- 예: "무엇을 남기고, 무엇을 지울 것인가"

**패턴 4: 선언**
- "[상태] [결과]"
- 예: "침묵은 새로운 금이다"

---

### The Manuscript 구조

```
[도입] (1-2문장)
───────────────────────────
경험의 단서. 구체적 묘사.
시각적 이미지 제시.

예: "외장하드 속 영상들. 파일명에 날짜만."

[전개] (3-4문장)
───────────────────────────
개인 → 보편 전환.
"우리"로 확장.

예: "우리는 끊임없이 기록한다.
하지만 돌아보지 않는다."

[심화] (3-4문장)
───────────────────────────
철학 연결.
본질적 질문 제기.

예: "공간 정리는 시간 정리다.
무엇을 남기고, 무엇을 지울 것인가."

[결말] (1-2문장)
───────────────────────────
열린 결론.
완결되지 않음.

예: "불완전한 기록들이 모여
한 사람의 초상이 된다."
```

```
□ 과장 없음? ("최고의", "완벽한")
□ 감정 과잉 없음? ("사랑해요", "대박")
□ 상업적 톤 없음? ("좋아요 구독")
```

## VII. 자기 점검 (Self-Check)

매 초고 작성 전 자문:

1. **"이 경험의 보편 키워드는 무엇인가?"**
   - 단어 3개 이내로 추출
   - 예: "시간", "기록", "불완전"

2. **"Aesop이라면 어떻게 썼을까?"**
   - Aesop 유사 테마 참조
   - 톤 비교

3. **"형용사가 과다한가?"**
   - 100자당 3개 이하
   - 과다 → 삭제 또는 동사로 전환

4. **"질문으로 끝나는가?"**
   - The Afterglow 확인
   - 답 제시했으면 → 삭제

5. **"72시간 내 제출 가능한가?"**
   - 완벽 추구 → 중단
   - MBQ 충족 → 제출
