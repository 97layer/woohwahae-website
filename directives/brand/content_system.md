---

## 1. 에세이 구조 — Hook-Story-Core-Echo

모든 Archive 에세이는 4단계 구조를 따른다.

### Hook (서두)
- 정의, 역설, 또는 질문으로 시작한다.
- 독자가 멈추게 만드는 한 줄.
- 설명하지 않고 선언한다.

### Story (서사)
- Opening: 개인 경험에서 시작
- Bridge: 개인 → 보편으로 전환
- Core: 핵심 관찰 또는 통찰
- 자기 연민이 아닌 관찰자 시점 유지

### Core (본론)
- 브랜드 해석 또는 개념 탐구
- 데이터/사례 기반. 주장에는 근거.
- 여백 있는 문단 — 문단 간 호흡.

### Echo (여운)
- 첫 문장(Hook)의 변주로 끝낸다.
- 결론 짓지 않는다. 여운을 남긴다.
- 독자가 자신의 답을 찾게 한다.

## 2. 길이 규칙

| 포맷 | 길이 | 용도 |
|------|------|------|
| 에세이 (Archive) | 5000-8000자 | 장문 브랜드 해석 |
| 캡션 (Instagram) | 50-100단어 | 일상 기록 |
| 리포트 (내부) | 제한 없음 | 분석 보고서 |
| 이슈 요약 | 200-300자 | 에세이 프리뷰 |

---

## 3. 이슈 발행 프로토콜

```
Issue = 하나의 브랜드 또는 하나의 개념에 대한 깊이 있는 탐구
```

### 이슈 구성
- **Cover**: 대표 이미지 (AD 검증)
- **Title**: 브랜드명 또는 개념명 + 서브타이틀
- **Body**: Hook-Story-Core-Echo
- **Metadata**: themes[], date, slug

### 이슈 소싱 경로
1. **Brand Scout**: 크롤링 → SA 분석 → CE 에세이 → AD 비주얼 → CD 승인
2. **개인 사유**: 텔레그램 신호 → 군집 성숙 → CE 에세이 → 동일 체인

## 4. 품질 게이트 (Ralph/QA)

모든 에세이는 발행 전 Ralph의 STAP 검증을 통과해야 한다.

### STAP 프로토콜
- **S (Stop)**: 완성 조건 명확화 — "이 에세이가 성공하려면?"
- **T (Task)**: CE 에세이 수신 → 기준 대조
- **A (Assess)**: 5-pillar 점수화 (각 20점, 총 100점)
- **P (Process)**: 70점+ 통과 → AD로 전달. 미달 → CE에 피드백.

### 5-Pillar 점수 기준

| Pillar | 평가 항목 |
|--------|-----------|
| 진정성 | 마케팅 언어 배제, 순호의 목소리 |
| 실용성 | 독자가 실천 가능한 인사이트 |
| 우아함 | 여백, 절제, 불필요한 수식 부재 |
| 정밀함 | 모호한 표현 부재, 근거 명시 |
| 혁신 | 관습적 시각 탈피, 새로운 관점 |

## 5. 카테고리 × 타입 매트릭스

### Layer A — 카테고리 (주제/층위 기준)

| 카테고리 | 정의 | 예시 주제 |
|----------|------|-----------|
| Seeing | 철학, 관점, 감각적 관찰 | 기다림, 물성, 내면, 시선 |
| Living | 라이프스타일, 공간, 일상 | 8평, 슬로우 라이프, 공간 |
| Working | 직업, 미용, 아틀리에 철학 | 시작, 머리카락, 일과 본질 |
| Making | 시스템, 브랜딩, 빌드 과정 | 72시간 규칙, 소음 제거 |
| Listening | 관계, 공명, 타인과의 접점 | 대화, 고객, 공명 |

### Layer B — 타입 (형식 기준)

| 타입 | 어체 | 구조 | 길이 |
|------|------|------|------|
| Essay | 한다체 | Hook-Story-Core-Echo, 독백, 관찰자 시점 | 300-800자 |
| Journal | 합니다체 | ~~~하는 법, 리드+본문+실천, 독자 유도 | 1200-3000자 |

### Essay 생성 규칙
- 한다체 사용. "~한다", "~이다", "~있다"
- 독백 형식. 관찰자 시점 유지. 독자에게 직접 말 걸지 않음.
- Hook → Story → Core → Echo 4단계 구조
- 마무리: 결론 없이 여운. 열린 질문 또는 여백.
- 금지: 강한 호소, 행동 유도, 독자를 향한 "당신"

### Journal 생성 규칙
- 합니다체 사용. "~합니다", "~입니다", "~됩니다"
- ~~~하는 법 / ~~~를 위한 안내 형식
- 리드(도입 요약) → 본문(단계별 설명) → 실천(온화한 제안)
- 독자를 향한 구체적 제안 가능. 독자가 실천 가능한 인사이트.
- 금지: 자기중심 관찰, 모호한 결론, 마케팅 언어

### SA → CE 분류 흐름
SA 분석 결과의 `content_category`(Seeing/Living/Working/Making/Listening) 필드를 CE가 수신.
CE는 corpus 주제와 신호 성격에 따라 `content_type`(essay/journal)을 결정.
기본값: essay.

## 6. 콘텐츠 파이프라인

```
Signal → [SA] 분석 → [Gardener] 군집 성숙 → [CE] 에세이
  → [Ralph] STAP 검증 → [AD] 시각 검증 → [CD] 최종 승인 → 발행
```

각 단계는 독립적으로 실패할 수 있다. 실패 시 이전 단계로 리턴.
