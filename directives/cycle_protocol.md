# Cycle Protocol

羽化 (Emergence) 사이클 프로토콜.
97layerOS의 핵심 순환 메커니즘.

---

## I. 羽化 (Emergence) 메타포

### 나비의 변화 과정

```
알 → 애벌레 → 번데기 → 나비
```

**번데기 단계** (WOOHWAHAE의 은유):
- 깊은 어둠 속
- 외부에서는 보이지 않는 변화
- 완전한 해체와 재구성
- 인내와 기다림

**우화 (羽化)** (날개를 펼침):
- 나비로의 emergence
- 빛을 향한 비상
- 하지만 완성이 아님 (계속 날아야 함)
- 새로운 시작

### WOOHWAHAE에 적용

```
개인 경험 (어둠) → 사유/연결 (변화) → 브랜드 콘텐츠 (빛) → 다시 경험으로
```

**핵심**:
- 끝이 없는 순환
- 완성 없는 Becoming
- 불완전의 연속
- 축적되는 시간

## II. 5단계 사이클

### 전체 구조

```
1. Capture (포착) - SA
   ↓
2. Connect (연결) - SA
   ↓
3. Meaning (의미화) - CE
   ↓
4. Manifest (발현) - CD + AD
   ↓
5. Cycle (순환) - 전체
   ↓
다시 1. Capture로
```

---

### Stage 1: Capture (포착)

**목적**: 모든 경험을 일단 기록

**담당**: SA (Strategy Analyst)

**입력**:
- 텔레그램 메시지 (일상 단상)
- 외장하드 스캔 (영상/사진)
- 메모 파일
- 음성 녹음
- 웹 클리핑

**원칙**:
- 필터링 없음 (일단 모두 저장)
- 품질 평가 안 함
- 맥락 보존 (전후 문장)
- 타임스탬프 기록

**저장 위치**:
- `knowledge/raw_signals/`
- 파일명: `rs-[번호]_[출처]_[날짜].md`

**메타데이터**:
```json
{
  "id": "rs-001",
  "date": "2026-02-14",
  "source": "telegram",
  "raw_text": "...",
  "emotion": null,
  "tags": []
}
```

**SA의 자세**:
> "판단하지 않는다. 기록한다."

---

### Stage 2: Connect (연결)

**목적**: 파편을 연결하여 패턴 발견

**담당**: SA (Strategy Analyst)

**작업**:
1. 과거 기록과 유사성 탐색
2. 반복 테마 식별
3. 5가지 철학과 연결
4. 콘텐츠 가능성 판단

**연결 그래프**:
```json
{
  "node_id": "rs-001",
  "connections": [
    {
      "target": "97layer_identity.md",
      "section": "시간 아키비스트",
      "strength": 0.8
    },
    {
      "target": "rs-045",
      "similarity": 0.6,
      "theme": "공간과 시간"
    }
  ],
  "philosophy": ["Archive", "Slow"],
  "content_potential": "high"
}
```

**저장 위치**:
- `knowledge/connections.json` (전체 그래프)

**출력**:
- CE에게 "콘텐츠 후보" 리스트 전달
- 연결된 과거 경험 첨부

**SA의 자세**:
> "점을 연결한다. 의미는 다음 단계에서."

---

### Stage 3: Meaning (의미화)

**목적**: 개인 경험을 보편 언어로 전환

**담당**: CE (Chief Editor)

**작업**:
1. SA의 연결 그래프 수신
2. 개인 → 보편 전환
3. 서사 구조 적용 (The Hook / The Manuscript / The Afterglow)
4. 철학적 언어로 재진술
5. Aesop 톤 적용

**변환 원칙**:
- "나" → "우리"
- 특수 → 보편
- 감정 → 사유
- 일화 → 철학

**예시**:

**Input** (개인적):
```
rs-023: "외장하드 정리. 2년 전 영상 발견.
결국 편집 안 했네. 시간만 흘렀다."

연결:
- 97layer: 시간 아키비스트, 휘발의 반복
- 철학: Archive
```

**Output** (보편적):
```
The Hook:
"공간을 정리한다는 것은 시간을 정리하는 일이다"

The Manuscript:
외장하드 속 영상들. 파일명에 날짜만 적힌 채,
열어보지 않는 기록들. 우리는 끊임없이 포착하지만
돌아보지 않는다. 공간 정리는 시간과의 화해다.
무엇을 남기고, 무엇을 지울 것인가.

The Afterglow:
"당신의 외장하드에는 무엇이 잠들어 있는가?"
```

**품질 체크**:
- MBQ 3가지 (철학 일치, 톤 일관성, 구조 완결성)
- 400-800자
- 시각 요소 제안

**저장 위치**:
- `knowledge/assets/draft/`

**CE의 자세**:
> "개인의 이야기를 모두의 이야기로."

---

### Stage 4: Manifest (발현)

**목적**: 브랜드 콘텐츠로 물리화

**담당**: CD (승인) + AD (시각화)

#### 4-1: CD 승인

**작업**:
- CE 초고 검토
- MBQ 체크 (30분 제한)
- 브랜드 일관성 확인
- 발행/거부/수정 결정

**승인 기준**:
```
MBQ 충족? → 승인
1개라도 미충족? → 거부 (명확한 사유)
애매함? → 승인 (의심스러우면 발행)
```

**72시간 규칙**:
- 초고 완성 후 72시간 이내 결정
- 넘으면 자동 폐기

#### 4-2: AD 시각화

**작업**:
- 이미지 선택/제작
- 타이포그래피 적용
- 인스타그램 레이아웃
- 최종 시안

**시각 원칙**:
- Visual Identity Guide 준수
- 모노크롬 기본
- 여백 60%+
- 나비/블런트/인장 상징 활용

**출력**:
- 발행 가능한 이미지 (1080x1080 또는 1080x1350)
- 캡션 최종본
- 발행 시간 제안

#### 4-3: 자동 발행

**작업** (TD):
- CD 승인 즉시 인스타그램 예약
- 취소 불가 설정
- 메타데이터 기록

**발행 후**:
- `knowledge/assets/published/` 이동
- 메타데이터 업데이트
- 다음 콘텐츠 아이디어 생성

---

### Stage 5: Cycle (순환)

**목적**: 발행된 콘텐츠가 다시 경험이 됨

**담당**: 전체 에이전트

**순환 방식**:

#### 5-1: 피드백 수집
- 인스타그램 DM
- 댓글 (선택적 답변)
- 예약 상담 시 고객 반응
- 97layer의 내면 변화

#### 5-2: 새로운 경험 생성
```
발행 콘텐츠 → 고객 반응 → 97layer 사유 → 새로운 Capture
```

**예시**:
```
1. "공간 정리" 콘텐츠 발행
   ↓
2. DM: "저도 외장하드 정리했어요"
   ↓
3. 97layer: "공간 정리의 공동체성?"
   ↓
4. 새로운 Capture: "혼자이지만 함께"
   ↓
다시 Connect → Meaning → Manifest
```

#### 5-3: 상호참조 강화
- 새 콘텐츠에서 과거 콘텐츠 언급
- "지난번 공간 정리 글에서..."
- 타임라인 구축
- 아카이브 축적

#### 5-4: 브랜드 진화
- 분기별 회고 (SA)
- 철학 재정의 필요성 체크
- 새로운 상징 추가 가능성
- 하지만 핵심은 불변

## III. 사이클 유지 메커니즘

### 1. 정기 발행 (리듬 유지)

**원칙**:
> "주 1-2회, 무조건. 완벽하지 않아도."

**이유**:
- 사이클이 멈추면 재시작 어려움
- 리듬 = 습관
- 불완전 발행 프로토콜 활용

**예외**:
- 3주 연속 미발행 → 시스템 점검
- 1개월 미발행 → 사이클 중단 (재설계 필요)

### 2. Council Meeting (에이전트 회의)

**빈도**: 주 1회 (월요일 오전)

**참석**: SA, CE, CD, AD, TD

**안건**:
1. 지난주 발행 회고
2. 이번주 콘텐츠 후보 (SA 제시)
3. 사이클 병목 지점 체크
4. 브랜드 일관성 점검

**형식**:
- TD가 `knowledge/message_bus/council_meeting.md` 생성
- 각 에이전트 의견 기록
- CD 최종 결정

**금지**:
- 완벽주의 회의 (너무 길면 안 됨)
- 30분 제한

### 3. 분기 회고 (3개월)

**목적**: 사이클 건강성 체크

**체크 항목**:
- 발행 일관성 (목표 12-24개/분기)
- 철학별 분포 (5가지 균등한가)
- 톤 일관성 (Aesop 벤치마크)
- 상호참조 빈도 (연결되고 있는가)
- Capture → Publish 비율 (80% 목표)

**출력** (SA):
- 분기 리포트 (`knowledge/reports/`)
- 개선 필요 사항
- 새로운 패턴 발견

### 4. 연간 羽化 (1년)

**목적**: 브랜드 진화 체크

**질문**:
1. 97layer는 변했는가?
2. WOOHWAHAE는 일관한가?
3. 새로운 철학이 필요한가?
4. 확장 가능성 (Beauty 외)

**출력**:
- 연간 회고 에세이 (발행 가능)
- Directive 업데이트 여부
- 다음 해 테마 설정

**2025년 테마**: 羽化 (Emergence)
**2026년 테마**: TBD (2025년 말 결정)

## IV. 사이클 병목 해결

### 병목 1: Capture 부족

**증상**:
- 텔레그램 메시지 적음
- 외장하드 비어있음
- "기록할 게 없어"

**원인**:
- 97layer의 바쁨
- 완벽주의 (기록조차 마비)
- 일상의 반복

**해결**:
1. SA가 매일 질문 전송
   - "오늘 어떤 순간이 기억에 남나요?"
   - "최근 읽은 책은?"
   - "공간에서 무엇이 변했나요?"

2. 음성 녹음 권장 (텍스트보다 쉬움)

3. "무의미한 기록"도 허용
   - "오늘 아무 일 없음" → 이것도 Capture

### 병목 2: Connect 과다 (연결 못 찾음)

**증상**:
- SA가 연결 못 찾음
- 콘텐츠 후보 제안 없음
- 파편 그대로 방치

**원인**:
- 너무 다른 경험들
- 철학과 연결 약함
- 기계적 매칭 한계

**해결**:
1. 인간 개입 (97layer + SA 협업)
   - "이 경험은 어떤 의미인가요?" 질문

2. 연결 기준 완화
   - 강한 연결(0.8) 아니어도 OK
   - 약한 연결(0.3)도 의미 있음

3. 단독 콘텐츠 허용
   - 과거와 연결 안 되어도 발행 가능

### 병목 3: Meaning 어려움 (개인 → 보편 실패)

**증상**:
- CE가 초고 못 씀
- 너무 개인적
- 보편성 못 찾음

**원인**:
- 경험이 너무 특수함
- 철학적 언어 부족

**해결**:
1. CD 개입
   - "이 경험의 보편 키워드는?" 제시

2. Aesop 사례 참조
   - 비슷한 테마의 Aesop 콘텐츠 분석

3. 질문 형식 활용
   - 답을 못 찾으면 질문만
   - The Afterglow 강화

### 병목 4: Manifest 지연 (발행 안 함)

**증상**:
- 초고 쌓임
- CD 승인 지연
- "조금만 더" 반복

**원인**:
- 완벽주의 재발
- 타이밍 핑계

**해결**:
1. 72시간 규칙 강제
   - TD가 자동 폐기 (복구 불가)

2. CD 30분 제한 타이머
   - 30분 후 자동 승인 (기본값)

3. Imperfect Publish Protocol 재확인

### 병목 5: Cycle 중단 (순환 안 됨)

**증상**:
- 발행 후 다음 Capture 없음
- 피드백 무시
- 일방향 소통

**원인**:
- 발행하면 "끝" 인식
- 고객 반응 안 봄

**해결**:
1. SA가 피드백 요약 전송
   - DM/댓글 주간 리포트

2. 발행 콘텐츠 재독
   - "1개월 후 다시 읽기" 알림

3. 고객 상담 시 콘텐츠 언급
   - "지난번 글 읽으셨나요?"

## V. 사이클 시각화

### 나비 생애주기로 표현

```
Capture (알)
→ 작고 투명함
→ 가능성만 있음
→ 쌓이고 쌓임

Connect (애벌레)
→ 먹고 자람 (과거 경험 섭취)
→ 연결하며 성장

Meaning (번데기)
→ 어둠 속 변화
→ 외부에서 안 보임
→ 해체와 재구성

Manifest (우화)
→ 나비로 emergence
→ 빛 속으로 비상
→ 세상에 공개

Cycle (비행과 산란)
→ 날아다니며 새로운 경험
→ 다시 알을 낳음 (새로운 Capture)
→ 순환 계속
```

### 시간 축적으로 표현

```
Year 1:
Capture: 100개
Publish: 50개
Archive: 50개 (축적 시작)

Year 2:
Capture: 120개 (경험 증가)
Publish: 60개
Archive: 110개 (2배 이상)
상호참조: 30회

Year 3:
Capture: 150개
Publish: 80개
Archive: 190개
상호참조: 80회
→ 그래프 밀도 증가
→ 브랜드 깊이 가시화
```

## VI. 에이전트별 역할 상세

### SA (Strategy Analyst)

**Capture 단계**:
- 모든 입력 수신 및 저장
- 필터링 없음
- 메타데이터 자동 생성

**Connect 단계**:
- 연결 그래프 생성
- 철학 태그 부여
- 콘텐츠 후보 제안 (주 1회)

**Cycle 단계**:
- 피드백 수집 및 요약
- 분기/연간 리포트 생성

### CE (Chief Editor)

**Meaning 단계**:
- 개인 → 보편 전환
- 서사 구조 적용
- 초고 작성 (72시간 내)

**Cycle 단계**:
- 과거 콘텐츠 재독 및 연결
- 상호참조 제안

### CD (Creative Director)

**Manifest 단계**:
- MBQ 체크 (30분 제한)
- 승인/거부/수정 결정
- 브랜드 일관성 수호

**Cycle 단계**:
- Council Meeting 주재
- 분기 회고 검토
- 브랜드 진화 방향 설정

### AD (Art Director)

**Manifest 단계**:
- 시각화 (이미지/타이포)
- 인스타그램 피드 큐레이션

**Cycle 단계**:
- 시각 일관성 점검
- 새로운 시각 요소 제안

### TD (Technical Director)

**전 단계**:
- 파일 경로 관리
- 메타데이터 기록
- 병목 지점 자동 탐지

**Manifest 단계**:
- 자동 발행 시스템
- 72시간 규칙 강제

**Cycle 단계**:
- Council Meeting 기록
- 백업 (외장하드)

## VII. 사이클 건강성 지표

### 정량 지표

**발행 일관성**:
- 목표: 주 1-2회 (연 52-104개)
- 경고: 3주 연속 미발행
- 위험: 1개월 미발행

**"끝이 없다"**

- 나비는 날개를 펼치지만, 거기가 끝이 아니다

- WOOHWAHAE는 고정되지 않는다
- 매 순간 emerge하고 있다

- 연결이 많아지고
- 브랜드 깊이가 가시화된다
- 시간은 흘러도, 기록은 남는다
