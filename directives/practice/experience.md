# Experience Map — 웹사이트 경험 설계

> **상위**: THE_ORIGIN.md §2.5 (3섹션 IA)
> **역할**: 3섹션 IA (About, Archive, Practice)와 기술 스택의 실무 규격.
> **권한**: PROPOSE

---

## 1. 사이트 구조 (IA)

```
woohwahae.kr/
├── about/          — 브랜드 서사 (철학, 이야기, 에디터)
├── archive/        — 매거진 (essay-NNN, magazine/, lookbook/)
├── practice/       — 수행의 실천 (아틀리에, 디렉션, 프로젝트, 프로덕트, 연락)
├── woosunho/       — 에디터 포트폴리오
└── lab/            — 실험 (네비 미노출)
```

네비: `Archive | Practice | About`

**WOOHWAHAE는 3섹션이 하나의 철학을 표현하는 다른 방식들이다.**

---

## 2. 섹션별 상세

### `/about/` — 철학의 선언 (Manifesto)

- Manifesto: "소음을 제거하면, 본질이 남는다."
- 1인칭 관점의 편집자 서사 및 반지하 8평 공간의 철학 전개.
- 내러티브 구조: 정체성 선언 → 일상의 리듬 → 사유의 확장
- 톤: 단정하고 묵직한 1인칭 서술. 구걸하지 않는 태도.
- 참조: THE_ORIGIN.md Part I (동아시아 렌즈, 3원칙)

### `/archive/` — 사유의 지층 (에세이)

- 사유 단위 발행 (Essay NN)
- 한 에세이 = 하나의 브랜드 혹은 한 가지 현상에 대한 깊이 있는 통찰.
- 5000자 내외의 밀도 높은 텍스트.
- 소싱: Brand Scout(관측) + 개인 사유(군집 성숙)
- 파일 구조: `website/archive/essay-{NNN}-{slug}/index.html`

### `/practice/` — 수행의 실천

- 서비스 랜딩 + 4가지 방식 (atelier, direction, project, product)
- 연락/예약 (contact.html)
- 톤: 정확하고 정중하게. 불필요한 친목 수식 배제.
- 참조: `practice/service.md`

### `/woosunho/` — 에디터 포트폴리오

- 순호의 작업물 30개. works/ 하위에 프로젝트별 폴더.
- 네비 미노출. about/editor.html에서 링크.

## 3. 네비게이션

```
[WOOHWAHAE]  Archive  Practice  About
```

- 고정 상단 네비게이션 (3섹션)
- 모바일: 햄버거 메뉴
- 현재 섹션 active 표시
- 최소주의: 드롭다운 없음, 깊이 1단계

---

## 4. 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 프론트엔드 | 순수 HTML/CSS/JS | 오버엔지니어링 방지, 속도, 통제권 |
| 프레임워크 | 없음 | 의도적 선택 |
| 반응형 | 모바일 우선 | 독자 환경 대응 |
| 배포 | GCP VM + nginx | 자체 호스팅, 통제권 |
| 자동 발행 | content_publisher.py → git push | 파이프라인 자동화 |

---

## 5. 레이아웃 원칙

- 화면의 60% 이상을 비워두는 여백의 우선 확보.
- 콘텐츠 최대 너비: 680px (텍스트 집중 영역), 960px (그리드 배열 영역).
- 색상/서체: `visual.md` 디자인 시스템 규격 참조.
- 이미지: 피사체에 조작을 가하지 않은 자연광과 그림자 중심의 시선 기록.

---

## 6. 금지된 경험 (Anti-Patterns)

- 독자의 시선을 빼앗는 팝업, 모달, 강제 구독 배너.
- 본질과 무관한 과도한 인터랙션 (좋아요 수 표기, 강압적 공유 유도).
- 조회수를 좇는 얕은 호흡의 콘텐츠 배치.
- 지저분한 외부 광고 삽입 (수익은 오직 본질적 가치 교환에서 창출).
- 맥락 없이 유행을 쫓는 레이아웃 변경.
