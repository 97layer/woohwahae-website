# 97LAYER Coding Style Guide

## 1. Syntax & Language

- **Language**: All comments and documentation must be in **Korean**.
- **Variables**: Use `snake_case` for variables and functions.
- **Classes**: Use `PascalCase`.
- **Constants**: Use `UPPER_SNAKE_CASE`.

## 2. Structure & Robustness

- **Error Handling**: Every major function must be wrapped in a `try-except` block.
- **Logging**: Use the standard `logging` library instead of `print()`.
- **Type Hinting**: Use Python type hints for all function signatures.

## 3. Aesthetics & Minimalism

- **Whitespaces**: Use 4 spaces for indentation. Keep vertical whitespace meaningful.
- **Comments**: Focus on "Why" rather than "What".
- **Imports**: Organize imports alphabetically. Group standard library, 3rd party, and local modules.
