# Imperfect Publish Protocol

완벽주의 마비를 해결하는 발행 프로토콜.
"불완전해도 괜찮다"를 시스템화한다.

---

## I. 문제 정의

### 97layer의 완벽주의 마비

**증상**:
- 눈은 높아짐 (다른 사람들의 매끈한 콘텐츠)
- 실력은 안 따라감
- 결과: **시도조차 못함**

**악순환**:
```
완벽 추구 → 실패 두려움 → 시도 안 함 → 성장 없음 → 완벽 더 멀어짐
```

**실제 사례**:
- 영상 → 외장메모리만 차지
- 글 쓰다가 → 문장 신경 쓰다 → 삭제
- "완성도 높은 결과물만 보여줘야 한다는 생각에 미루기"

**2025년 목표**:
> "미완성이어도 괜찮다. **일단 하기**"

---

## II. 프로토콜 철학

### 핵심 원칙

**"완벽이 아닌 브랜드 최소 기준"**

- 완벽 = 도달 불가능
- 브랜드 최소 기준 = 명확히 정의 가능
- 기준 충족 = 발행
- 완벽 추구 = 선택 사항

**Aesop 사례**:
- Aesop도 모든 콘텐츠가 완벽하지 않다
- 하지만 "Aesop다움"은 일관적이다
- 우리는 "완벽한 Aesop"이 아닌 "일관된 WOOHWAHAE"

**불완전의 미학**:
> "불완전함이야말로 삶의 가장 진솔한 기록" (97layer)

---

## III. 브랜드 최소 기준 (MBQ: Minimum Brand Quality)

### 필수 요건 (3/3 충족 시 발행)

#### 1. 철학 일치
- [ ] 5가지 철학 중 1개 이상 명확히 연결
- [ ] 97layer 정체성과 모순 없음
- [ ] WOOHWAHAE 금기 사항 위반 없음

#### 2. 톤 일관성
- [ ] 지적이고 절제된 언어
- [ ] 과장/감정 과잉 없음
- [ ] 상업적 톤 없음

#### 3. 구조 완결성
- [ ] The Hook 존재 (1문장)
- [ ] The Manuscript 존재 (본문)
- [ ] The Afterglow 존재 (질문)

**이 3가지만 충족하면 발행한다.**

---

### 선택 요건 (없어도 됨)

다음은 **있으면 좋지만 없어도 발행 가능**:

- 이미지 완벽한 조명/구도
- 800자 (400자여도 OK)
- 과거 콘텐츠 상호참조
- 시각 요소 (텍스트만도 OK)
- 타이밍 최적화
- 완벽한 문장 (약간 어색해도 OK)
- 맞춤법 100% (오타 1-2개 OK)

**"충분히 좋음 (Good Enough)"이 기준이다.**

---

## IV. 3단계 발행 프로세스

### Phase 1: 72시간 규칙

**원칙**:
> "콘텐츠 초고 완성 후 72시간 이내 발행 결정"

**이유**:
- 72시간 넘으면 완벽주의 침투
- "조금만 더 다듬고..." → 무한 지연
- 신선도 유지 (경험과 콘텐츠 간격 최소화)

**작동 방식**:
```
Day 0: CE 초고 완성
Day 1: CD 검토 (MBQ 체크)
Day 2: AD 시각화
Day 3: 발행 또는 폐기 결정
```

**폐기 조건**:
- MBQ 3가지 중 1개라도 미충족
- 97layer 본인이 거부 (진정성 부족)
- 타이밍 부적절 (1주일 이내 재검토)

### Phase 2: CD 승인 (30분 제한)

**원칙**:
> "CD는 30분 이내에 승인/거부/수정 결정"

**이유**:
- 오래 보면 흠 찾기 시작
- 직관 신뢰 (첫 인상이 정확함)
- 병목 방지

**승인 기준**:
- MBQ 3가지 충족? → 승인
- 1개라도 미충족? → 거부 또는 수정 (명확한 방향 제시)
- 애매함? → 승인 (의심스러우면 발행)

**금지**:
- "조금만 더 다듬으면..." (무한 루프)
- "이미지를 바꿔볼까..." (선택 요건)
- "다음 주로 미루자" (72시간 규칙 위반)

### Phase 3: 자동 발행 (예약)

**원칙**:
> "승인 즉시 발행 예약 (취소 불가)"

**이유**:
- 승인 후 재고하면 완벽주의 재발
- "발행 버튼" 누르는 순간의 두려움 제거
- 시스템이 대신 결정

**작동 방식**:
1. CD 승인 → 즉시 인스타그램 예약 (Meta API)
2. 예약 완료 알림
3. 취소 불가 (시스템적 차단)
4. 발행 후 메타데이터 자동 기록

**예외**:
- 97layer 본인 거부권 (승인 후 1시간 이내, 월 1회만)
- 명백한 오류 발견 (오타 아닌 사실 오류)

### 시나리오 1: "문장이 어색해"

**CE 대응**:
```
MBQ 체크:
- 철학 일치? ✅
- 톤 일관성? ✅ (어색해도 톤은 맞음)

```

**CE 답변**:

### 시나리오 2: "이미지가 별로야"

**97layer 반응**:
> "사진이 흐릿한데... 다시 찍을까?"

**AD 대응**:
```
Visual Identity Guide:
- 완벽한 선명도 필요? ❌ (선택 요건)
- 흐릿함 = 불완전 허용 ✅
- 브랜드 톤 유지? ✅ (모노크롬, 여백)

→ 발행 가능
```

**AD 답변**:
> "흐릿함이 오히려 '시간의 흔적' 느낌을 줍니다. WOOHWAHAE는 완벽한 사진을 추구하지 않습니다."

### 시나리오 3: "더 좋은 아이디어가 떠올랐어"

**97layer 반응**:
> "지금 쓴 건 별로고, 이렇게 쓰면 더 좋을 것 같은데..."

**CD 대응**:
```
72시간 규칙:
- 초고 완성 후 48시간 경과
- 완벽주의 침투 징후

→ 즉시 발행 + 새 아이디어는 다음 콘텐츠로
```

**CD 답변**:
> "좋은 아이디어는 다음 콘텐츠로 기록해두겠습니다. 지금 것은 이미 충분합니다. 발행합니다."

### 시나리오 4: "조회수가 낮을 것 같아"

**97layer 반응**:
> "이런 긴 글은 아무도 안 읽을 텐데..."

**CD 대응**:
```
WOOHWAHAE 원칙:
- 조회수 추구? ❌
- 알고리즘 저항 ✅
- 깊이 축적 ✅

→ 발행
```

**CD 답변**:
> "우리는 조회수를 추구하지 않습니다. Aesop도 긴 텍스트를 허용합니다. 읽는 사람은 소수여도 깊게 읽을 것입니다."

## VI. "일단 하기" 3단계

### 1단계: 포착 (Capture)

**목표**: 완벽하게 기록하지 말고, **일단 기록**

**방법**:
- 텔레그램에 단상 메시지 (문장 완성 안 해도 됨)
- 음성 녹음 (말 더듬어도 됨)
- 사진 찍기 (흐릿해도 됨)
- 메모 (키워드만)

**금지**:
- "나중에 정리해서 기록해야지" → 영원히 안 함
- "이 정도로는 부족해" → 파편도 가치 있음

**SA 역할**:
- 모든 입력 무조건 저장
- 품질 평가 안 함
- "기록했습니다" 확인만

### 2단계: 연결 (Connect)

**목표**: 완벽한 서사를 만들지 말고, **일단 연결**

**방법**:
- 과거 기록과 유사성 발견
- "이것과 저것이 비슷한데?" 수준
- 완벽한 해석 아니어도 됨

**금지**:
- "더 깊은 의미를 찾아야 해" → 표면적 연결도 가치 있음
- "이건 너무 뻔한 연결인데" → 뻔함도 OK

**SA 역할**:
- 기계적 매칭 (키워드, 날짜, 감정)
- 해석 최소화
- "이것들이 연결됩니다" 제시

### 3단계: 발행 (Publish)

**목표**: 완벽하게 다듬지 말고, **일단 발행**

**방법**:
- MBQ 3가지만 체크
- 72시간 규칙 엄수
- CD 승인 후 자동 예약

**금지**:
- "조금만 더" → 영원히 미완성
- "이번엔 완벽하게" → 완벽은 없음
- "타이밍을 기다려" → 지금이 타이밍

**CD 역할**:
- 30분 이내 결정
- "충분히 좋음" 기준
- 의심스러우면 발행

## VII. 완벽주의 탐지 및 차단

### 완벽주의 패턴 탐지

**SA가 다음 패턴 감지 시 경고**:

1. **무한 수정**:
   - 같은 콘텐츠 3회 이상 수정 요청
   - → 경고: "이미 충분합니다. 발행하세요."

2. **과도한 비교**:
   - "다른 브랜드는..." 멘트
   - → 경고: "우리는 WOOHWAHAE입니다. 비교하지 마세요."

3. **타이밍 핑계**:
   - "다음 주에", "좋은 날에", "준비되면"
   - → 경고: "지금이 타이밍입니다. 72시간 규칙."

4. **자기 검열**:
   - "이건 너무 개인적인데", "사람들이 관심 없을 것 같은데"
   - → 경고: "개인적인 것이 진정성입니다."

### 시스템적 차단

**TD가 다음 설정 구현**:

1. **수정 횟수 제한**:
   - 콘텐츠당 최대 2회 수정
   - 3회째 수정 요청 시 강제 발행 또는 폐기 선택

2. **시간 제한**:
   - 초고 완성 72시간 후 자동 폐기
   - 복구 불가 (새로 쓰도록 강제)

3. **취소 불가**:
   - CD 승인 후 발행 예약 취소 버튼 비활성화
   - 97layer 거부권도 1시간 제한

## VIII. 불완전 허용 사례

### 허용 사례 1: 오타

**상황**: "있다" → "잇다" 오타
**판단**: 1-2개 오타는 OK (사실 오류 아님)
**이유**: 완벽한 맞춤법보다 진정성

### 허용 사례 2: 문장 어색

**상황**: "그것은 마치 나비가 되는 것처럼 같았다" (중복)
**판단**: 톤만 맞으면 OK
**이유**: 어색함도 97layer의 언어

### 허용 사례 3: 이미지 흐릿

**상황**: 자연광 사진, 약간 흐릿함
**판단**: OK (오히려 빈티지 느낌)
**이유**: 완벽한 선명도 불필요

### 허용 사례 4: 짧은 글

**상황**: 400자 (기준 800자)
**판단**: OK (The Hook + The Afterglow만 있어도 됨)
**이유**: 길이보다 밀도

### 허용 사례 5: 시각 요소 없음

**상황**: 텍스트만 있는 카드
**판단**: OK
**이유**: 타이포그래피도 시각 요소

---

## IX. 발행 후 관리

### 수정 금지 원칙

**원칙**:
> "발행 후 수정하지 않는다"

**이유**:
- 수정 = 완벽주의 재발
- 불완전도 기록의 일부
- 시간의 흔적 (당시의 생각)

**예외** (매우 드물게):
- 사실 오류 (날짜, 이름 등)
- 링크 오류
- 법적 문제

### 회고는 하되 수정은 안 함

**분기별 회고** (3개월마다):
- 지난 콘텐츠 리뷰
- 패턴 발견
- 개선 방향 설정

**하지만**:
- 과거 콘텐츠 수정 금지
- "그때는 저랬구나" 인정
- 다음 콘텐츠에 반영

**SA가 회고 리포트 생성**:
- 철학별 분포
- 톤 일관성 분석
- 불완전했지만 발행한 것들의 가치

**DO**:
- 모든 입력 무조건 저장

**DON'T**:
- 품질 평가
- "이건 부족한데..." 멘트

### CE (Chief Editor)

**DO**:
- MBQ 기준만 체크
- 72시간 규칙 엄수
- "충분히 좋음" 판단

**DON'T**:
- 완벽 추구

**DO**:
- 30분 이내 결정

- "다음 주로" 미루기

## XII. 성공 지표

### 정량 지표

**발행 일관성**:
- 목표: 주 1-2회 (연 52-104개)
- 실패: 3주 연속 미발행
- 성공: 12주 연속 발행

**폐기율**:
- 목표: 20% 이하 (80% 발행)
- 의미: 초고의 80%는 발행 가능 수준

**수정 횟수**:
- 목표: 평균 1회 이하
- 의미: 초고가 이미 충분함

### 정성 지표

**97layer 피드백**:
- "생각보다 쉽게 발행되네"
- "부담이 줄었어"
- "일단 하게 됐어"
