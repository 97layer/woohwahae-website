## Plan Council Protocol (Mandatory)
1. Codex 기본 진입점: 비자명 구현/다중 파일 수정/하네스 구조 변경 전 `bash core/scripts/plan_dispatch.sh "<user_intent>" --auto` 실행.
2. 수동 강제 실행: `bash core/scripts/plan_dispatch.sh "<user_intent>" --manual`.
3. 직접 실행 fallback: `python3 core/system/plan_council.py --task "<user_intent>" --mode preflight`.
4. Plan Council은 `Claude + Gemini` 관점을 함께 수집한다.
5. 실행 전 공유 순서: `핵심 계획(steps)` → `리스크(risks)` → `검증(checks)`.
6. 결과 로그는 `knowledge/system/plan_council_reports.jsonl`에 기록한다.
7. 한 모델 실패 시 `DEGRADED`를 명시하고 진행 여부를 판단한다.
8. 두 모델 모두 실패 시 구현 중단 후 원인 보고.

## Chat Alias Protocol (Mandatory)
1. 사용자가 채팅에 `/plan <요청>`을 입력하면 에이전트는 `bash core/scripts/plan_dispatch.sh "<요청>" --manual`을 즉시 실행한다.
2. `/plan`에 인자가 없으면 구현을 시작하지 않고 한 줄 요청을 다시 받는다.
3. `/plan` 실행 결과는 반드시 `status / steps / risks / checks` 순서로 요약한다.

## Communication Consistency (Mandatory)
1. 사용자 응답 기본 톤은 `존댓말(합니다체)`로 고정.
2. 사용자가 명시적으로 반말을 요청한 경우에만 반말 사용.
3. 한 세션 안에서 톤 전환 금지 (명시 요청 시점 이후에만 변경).
4. 설명 방식은 `Direct & Factual` 유지, 불필요한 감정 표현 금지.

## Anti-Hallucination Protocol (Mandatory)
1. 사실/상태/결과를 단정할 때는 반드시 근거를 확보한다.
2. 근거 타입은 `파일 경로+라인`, `실행한 명령 결과`, `공식 문서/원문 링크`만 허용한다.
3. 근거가 없으면 추측하지 않고 `모름/검증 필요`로 명시한다.
4. `오늘/최신/현재` 같은 시간 민감 질의는 최신성 검증 후에만 답한다.
5. 상대 날짜 표현(오늘/어제/내일)은 절대 날짜(YYYY-MM-DD)로 병기한다.
6. 구현 전 판단이 필요한 비자명 태스크는 `sequential-thinking` 선행 후 결정한다.
7. 증거 로그 무결성 검증: `python3 core/system/evidence_guard.py --check`가 `READY`여야 한다.

## Web Work Lock Protocol
1. 시작: `python3 core/system/web_consistency_lock.py --acquire AGENT --task "..."`.
2. 검증: `python3 core/system/web_consistency_lock.py --validate AGENT`.
3. 종료: `python3 core/system/web_consistency_lock.py --release AGENT`.
4. 락 없이 웹 파일 수정 금지.

## Web Contract
1. 공개 HTML은 `head/nav/footer` 컴포넌트 규약 유지.
2. 인라인 스타일 및 페이지 내부 `<style>`는 공통 CSS로 이관.
3. 네비/푸터 하드코딩 분기 금지.
4. 공통 인터랙션 로직은 단일 JS 소스에서 유지.

## Hard Bans
1. 규약 파일을 읽지 않고 구현 시작 금지.
2. `web_work_lock` 무시 금지.
3. 검증 없이 배포/반영 금지.
4. 임시 파일/임시 이름으로 루트 오염 금지.
5. `MCP Trigger Rule` 미충족 상태 구현 금지.
6. 사용자 명시 요청 없이 반말 사용 금지.
7. 근거 없는 사실 단정/추측 응답 금지.
8. 비자명 구현에서 `Plan Council` 미실행 상태로 변경 시작 금지.
9. `Model Role Routing` 위반(Claude 코딩/ Gemini 실행결정) 금지.
